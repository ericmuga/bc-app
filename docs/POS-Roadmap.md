`ericmuga/prime-pos` and `ericmuga/order-invoice-service` (both share this app's Vue/Express + mssql stack, so ports are mechanical)._

## Phase 1 — Payments foundation _(highest value; most already part-built)_
1. **Bank Deposit payment method** — rename the existing `deposit` method → **"Bank Deposit"**, ensure a **reference** field. _New (small)._ — Q5
2. **Reference on payments** — `PosPayment.Reference` already exists and checkout/confirm persist it; extend the checkout UI so every method (not just M-Pesa) can capture a reference. _Mostly done._
3. **Per-method transaction lists (DB or API)** — for each payment method, a maintained transactions table with **upload/download** (CSV/XLSX) for offline use, and a per-method toggle **Read from DB | Read from API**. _New._ — Q1
4. **Self-contained payments fetch API** — port the `mpesa_transactions` fetch from `order-invoice-service` into bc-app so it no longer depends on the external app; **add shop till** (`BusinessShortCode`) to the returned payload (bc-app's `fetchMpesaLocal` already filters by till). **Index/optimization guidance below.** _Port._
5. **Cross-till approval** — when an M-Pesa code search returns a payment whose till/paybill ≠ the current shop's short code, require **approval by the owning shop** before it can be applied. _New (workflow)._ — Q2
6. **Till/Paybill paid-to at checkout & receipt** — show and print which till/paybill the payment went to. _Port from prime-pos._

## M-Pesa fetch optimization guidance (item #4)

`mpesa_transactions` is queried by **amount**, **till (BusinessShortCode)**, **time window**, and **TransID (code)**. Recommended:

- **Composite index for the amount lookup:**
  `CREATE INDEX IX_mpesa_amount_till_time ON dbo.mpesa_transactions (TransAmount, BusinessShortCode, TransTime DESC) INCLUDE (TransID, MSISDN, FirstName, MiddleName, LastName, is_claimed);`
- **Code lookup:** `CREATE INDEX IX_mpesa_transid ON dbo.mpesa_transactions (TransID);`
- **Time-window scans:** `CREATE INDEX IX_mpesa_transtime ON dbo.mpesa_transactions (TransTime DESC);`
- **Query-string params to add** for narrower searches / faster filters: `till` (BusinessShortCode), `since`/`hours` (bound the time window — already used), `claimed` (exclude already-applied), `msisdn` (last-4 phone), `code` (TransID), and `limit`. Always bound by `TransTime` to keep the index seek selective.
- Mark applied payments (`is_claimed`) and filter them out by default to shrink result sets.

## Decisions (2026-06-27)
- **Build order:** _Quick wins first_ — Bank Deposit rename + reference, item photos, stock-request RF column, till/paybill on receipt, coupons port. Then Phases per priority.
- **WHVAT:** configurable base (% of taxable value **or** % of VAT only) in setup; **PIN required** for withholding customers (block checkout without PIN).
- **AI insights:** target **Microsoft Copilot** integration (not Claude API) — prepare/scaffold the integration; defer the build.
- **Customer display:** **separate networked device** (subscribes over the network to the active order).

## Open questions (blocking design)

- **Q1 (payment lists):** For non-M-Pesa methods, what's the canonical transaction source/shape (columns), and where do uploaded files live — a new per-method table in BCApp, or the orders DB? Is the DB/API toggle per method global, or per shop?
- **Q2 (approval & tickets):** Preferred workflow/notification channel — in-app notifications, email, or both? Who approves a cross-till claim (owning shop's cashier, or an admin)?
- **Q3 (discounts):** Should the ported discount engine **replace** the existing `PosSpecialPrice` offers, or run alongside? Discount precedence when item, category, and contact rules all match?
- **Q4 (WHVAT):** Confirm direction — WHVAT reduces what the customer pays now by 2% of taxable value and FC reclaims later (standard KRA). The "% of total vs % of VAT only" toggle: keep both options, or fix to the legal 2%-of-taxable-value? Customer PIN: block checkout if a withholding customer has no PIN?
- **Q5 (Bank Deposit):** Is `deposit` currently a payment **type code** or a payment **class**? Rename in place (keep code) or new code `BANKDEPOSIT`?
- **Q6 (Offers vs Special Prices):** Offers overlaps the existing date-bound `PosSpecialPrice`. Build Offers as a new buttons feature, or extend Special Prices to drive the offer buttons?
- **Q7 (item photos):** Where are images stored in prime-pos (DB blob, URL, or file store)? Same approach here, or upload to this app?
- **Q8 (customer display):** Second **browser window/route** on the same PC (synced via BroadcastChannel/WebSocket), or a separate networked device?
- **Q9 (WhatsApp):** Which provider — Meta Cloud API (templates) — and do you have a verified WABA + phone number ID + token? Templates pre-approved?
- **Q10 (AI insights):*use copilot api to be integrated later 
- **Q11 (manual):** Do you have Farmers Choice brand assets (logo, colors, fonts) to embed, or should I pull them from the app's current theme?
